# Eighth-Iteration Field Trial — Steward Log

**Date:** 2026-07-02
**Package version:** 0.7 (post-seventh-iteration, commit ad7334f)
**Trial protocol:** Build a small world end-to-end using only `docs/worldbuilding-system/*`, following the README "For a new world" path literally: seed kernel → decomposition → admission → propagation → one constraint composition → one contradiction repair → one protected effect under the opacity accountability test → QA regression profile.
**Steward stance:** No authorship stake in the world. The world is invented fresh for this trial. Every point of friction, every instrument skipped (with reason), and every question the documents failed to answer is logged here. Log entries are numbered F-## (friction), S-## (skip), Q-## (unanswered question), V-## (things that worked well / valuable observations).

**Logging discipline:** entries are written at the moment they occur, in pipeline order. Verdicts and recommendations are only at the end, after QA.

## Stage 0 — Entry and reading path

- **V-01.** The README's "For a new world" nine-step path is a genuinely usable trial script: it names a concrete file at every step. I never had to guess what to open first.
- **F-01.** The "For a new world" path never tells the steward to read `02_world_model.md` or `03_truth_layers_and_canon_governance.md`, but the admission protocol (step 5) presupposes their vocabulary (truth layers, canon statuses, constraint tags). The "For an existing world" path *does* assign 02/03 as reading. A first-time steward following the new-world path literally will hit `06` needing terms they were never routed through. Either add 03 (at least) to the new-world reading list or state that 06 is self-contained enough without it (to be verified when I reach 06).
- **V-02.** `01_core_theory.md` reads as doctrine should: the 12 laws are short, the four anti-failure questions and the craft ceiling ("stop when answers become decorative") give a working steward an actual stopping rule, not just exhortation.

## Stage 1 — World kernel (step 2)

Minor, but I had to interpret rather than follow.
- **Q-01.** The template says "List the few seed facts" — no guidance on how many is too many, or what happens if a premise sentence smuggles several facts (mine does: SF-1 contains at least capability + source + consumption rule). Presumably `05` decomposition handles this; noting to verify. *(Resolved at Stage 2: `05` Phase 2 handles it cleanly, and the thin-start rule gives the "too thick" test. The kernel template could cross-reference the thin-start rule, but this is minor.)*

## Stage 2 — Decomposition and frontloaded seed audit (steps 3–4)

Artifacts: `reports/field-trial-world/seed_decomposition.md`, `reports/field-trial-world/frontloaded_seed_audit.md`. 4 kernel facts → 15 atomic seeds.

- **F-04.** `05` Phase 1 defines the kernel as nine named items (Essence, Aesthetic promise, Starting scale, Reality and consequence mode, Primary pressures, Forbidden easy answers, Mystery posture, Initial seed facts, Ordinary-life anchor) and then says "Use `templates/world_kernel.md`" — but the template's sections don't match that list. The template has no "Starting scale," no "Primary pressures" (only "Initial domains of pressure," which is a different question — forces vs. affected domains), and its "Ordinary-life promise" (residue) is not `05`'s "Ordinary-life anchor" (one named ordinary person). Because the README has the steward fill the template at step 2 *before* reading `05` at step 3, I discovered the mismatch only after my kernel was written, and had to decide whether to retrofit. I did not retrofit (the audit passed without those items), but one of the two lists should be the single source of truth. Concretely: I never named an ordinary-life anchor *person*, and nothing downstream ever asked for them again — the anchor person exists only in `05` Phase 1 and silently drops out of the instrument chain.
- **F-05.** README step order (kernel → decompose → audit → admit) vs. `05` Phase 3, which folds the audit into admission ("Use `06...` and `checklists/frontloaded_seed_audit.md`"). Not contradictory, but two different placements of the same instrument; a steward wondering "have I done this at the right time?" gets no answer. Suggest the checklist itself state when it runs (after decomposition, before first admission).
- **F-06.** The audit requires "Each seed fact has an initial status" *before* admission — but status is assigned *by* admission. The only coherent pre-admission answer is "proposed," which makes the checkbox nearly information-free. Either the box should say "status = proposed confirmed" or it should ask for the *intended* status so the gate can check the intention.
- **F-01 (confirmed).** The audit's inventory section ("truth layer," "initial status," "constraint tags separated from statuses") required `03`'s vocabulary at README step 4, two steps before the path routes a steward anywhere near `03` — and in fact the new-world path *never* assigns `03` as reading. I had to leave the path and read `03` in full to fill three checkboxes. `03` was worth reading in full (see V-04), which makes its absence from the new-world reading list stranger.
- **V-04.** `03` is the strongest document I have read so far: the label-separation table, the operation-context matrix, and the decision-jurisdiction table each answered a question within minutes of my forming it (e.g., "is 'contested' a status or a layer?" — answered by the table; "does S4d's plural belief ecology need statuses per belief?" — answered by layer/status separation). Canon-debt categories immediately gave me the right words for the two debts my audit surfaced (propagation debt on harvest volume, evidence debt on Lee Shore holdings).
- **V-05.** The audit's "What latency or path dependence explains why the world is not already different?" question did real work: it forced me to notice that for a centuries-old capability the burden *inverts* — the world must be shown already-different, which becomes a propagation shopping list rather than a plausibility problem. The checklist caused a genuine design insight, which is what an instrument is for.
- **Q-02.** Decomposition guidance gives one worked example (dead → echoes) but no target granularity: is "S1d — one taster, once" one seed or three (dissolution medium, perspective fidelity, consumption)? I chose by feel. A one-line rule ("split until each seed could be independently rejected without destroying its siblings") would have removed the hesitation. My 15 seeds each pass that test, but I derived the test myself.

## Stage 3 — Seed admission (step 5)

Artifacts: `admission_ledger.md`, `capability_card_memory_salt.md`, `fact_cards.md` (4 cards), `mystery_ledger_weeping_shelf.md`.

- **F-07.** For a capability, `06` instructs using *both* the canon fact card and the capability card — and the two templates overlap on roughly half their sections (costs, countermeasures, institutions, economy, distribution/scope, residue). Nothing says how to divide labor between them. I invented a division (fact card = governance record + shock-cone; capability card = causal analysis) and cross-referenced, but a steward less willing to improvise would write the same content twice. One sentence in `06` or on the templates ("when a capability card exists, the fact card may reference it for sections X, Y, Z") would fix this.
- **F-08.** The card template's "record one fact, not a bundle" rule collides with decomposition output: 15 atomic seeds would mean 15 cards, but severity-scaled evidence says minor facts owe only "precise statement, scope, truth layer, status, one consequence check" — and *no instrument exists for that record*. I invented an admission-ledger table (one row per minor seed). It worked well, which suggests the package genuinely wants a lightweight batch-admission instrument (or an explicit statement that minor-work facts need no card and where their five required items should live).
- **F-09.** The gate checklist asks for "Admission decision operation recorded" in the singular, but real admissions compose operations: SF-1 was constrain + price + institutionalize in substance. `03` lists these as distinct operations without saying whether an admission may record several. I recorded a primary operation with the others as context; the docs are silent on whether that is correct.
- **Q-03.** `05`'s seed admission order ends with "aesthetic promises," but no admission instrument targets an aesthetic rule — the fact card's type list includes "aesthetic," yet every gate question is about consequence machinery. I deferred the kernel's aesthetic promises to the aesthetic pass (step 7) without any document telling me that was the right disposal. Will report whether `17` catches them.
- **V-06.** The severity two-scale mapping paragraph in `06` (levels classify the fact, work scale classifies the owed work, with an explicit level→work map) resolved every "how much work does this owe?" question I had. The outlook flagged severity-scale unification as a candidate fix; on this evidence the seventh iteration's mapping paragraph already did the job.
- **V-07.** The gate is a real gate, not a ritual: its "Contradiction risks checked" box surfaced the court-testimony-versus-consumption tension *at admission time*, and resolving it produced the world's best institution (the Single Witness doctrine). Second instrument-caused design insight of the trial (after V-05).
- **V-08.** The mystery ledger template is the best instrument in the package. Its separation of *author-secret* from *deliberately undecided* is what made the trial's strongest design choice visible (committing to an **empty** author-secret set — deciding is itself the breach). The sacred-opacity accountability block forced six specific answers I would not have volunteered, and the "recurrence/boundary rule" prompt yielded the operative principle that invariance is what makes a sacred effect auditable. Zero friction in the template itself.

## Stage 4 — Propagation (step 6)

Artifact: `propagation_report_SF1.md` (full 14-domain pass, sweep run, contradiction check).

- **V-09.** The propagation mechanisms and branch types are generative, not just descriptive: asking the black-market-branch questions against the one-taster constraint produced the world's most distinctive economy (stolen grains are spent secrets, so the real theft economy is extortion — "I have your father's memory"). That insight is directly attributable to the instrument.
- **V-10.** The stop-when-governed rule (every high-pressure consequence answered / scoped out / assigned as debt / protected) is a real, checkable stopping condition. I used all four states and knew when I was done — the single hardest problem in propagation, solved by one sentence.
- **V-11.** Domain-atlas triage (direct/dependency/reaction, domain minimum, negative domains) kept the 14-domain pass proportionate; the "declared quiet" convention let me skip ecology without guilt, and the negative-domain example in `04` ("cheap resurrection but unchanged inheritance law") was almost exactly my world's biggest risk — good calibration.
- **V-12.** The sweep's mandatory contradiction check caught a genuine self-inflicted conflict: S2b (uncertified possession finable) vs. S3b (pre-war Lee Shore salts legal to keep) — pre-war salts cannot be certified, so they were simultaneously legal and finable. I did not plant this; the admission wording created it and the instrument caught it. Third instrument-caused catch of the trial.
- **F-10.** The shock cone is demanded by three instruments at three granularities: fact card ("Shock-cone summary"), propagation report ("Shock-cone order"), sweep checklist ("Shock cone" boxes). For SF-1 I effectively wrote it three times. The report is obviously the master record; one sentence in `06`/`07` ("when a propagation report exists, the fact card's shock-cone summary is a digest of or pointer to it") would eliminate the duplication anxiety.
- **F-11.** Vocabulary slip: the propagation report template's field is labeled "Severity **level**" but offers the *work-scale* values (minor/moderate/major/foundational/catastrophic), while `07` triggers the sweep on "Level 2 or higher" (the *levels* scale). `06`'s mapping paragraph disambiguates — but only if the steward has it fresh in mind. Rename the template field "Severity (work scale)" or similar.

## Stage 5 — Constraint composition

- **V-14 (part 1).** The constraint budget (access/cost/failure/social/loophole, "too many unrelated constraints feel patched") is a genuinely useful design-review tool: my five constraints all derive from two metaphysical limits plus one institution, and the budget check is what made me verify that derivation.

## Stage 6 — Contradiction repair

Artifact: `contradiction_report_C1.md`. C-1 (pre-war salts simultaneously legal and finable) repaired via clarify scope + institutionalize (the Concession Rolls), disposition recorded, repair propagated.

- **V-14 (part 2).** The admission-operation vs. repair-operation jurisdiction boundary — which earlier iterations reportedly struggled to keep separated — held up under field conditions. At C-1 I knew without hesitation which vocabulary governed, and the contradiction report's note ("if the repair creates a new proposal, record that separately") correctly forced the new Concession Rolls institution back through *admission* rather than letting it ride in on the repair. The `03`/`13` split is learnable and load-bearing.
- **F-12.** The contradiction report's "Repair operation" field is singular, but real repairs compose (mine: clarify scope + institutionalize; `13` itself says repairs must be propagated and a constraint fix "may create new institutions"). Same singular-field problem as F-09 on the admission side. Both templates should say "operation(s), primary first."
- **F-13.** **The card-lifecycle gap — my most important structural finding.** The repair superseded S2b with reworded S2b′. Nothing in the package says where the corrected wording *lives*: edit the original fact card in place? Append? New card? The original card for S2b now sits in my records stating the wrong law, corrected only by a contradiction report filed elsewhere. `03` supplies the statuses (superseded, deprecated) but no instrument describes card upkeep, and no instrument provides a **current-state view** of canon after repairs accumulate. Over a real world's lifetime this compounds: a steward asking "what is the law *today*?" must replay every contradiction report against every card. The package governs *changes* meticulously and *state* not at all. Even one paragraph of doctrine ("the fact card is living and is updated in place; superseded wording moves to the card's history section; reports are the audit trail, cards are the present tense") would close it.
- **Q-04.** `13`'s weak-mystery list includes "it exists only because the author has not decided," while mystery type 8 (authorial mystery boundary) blesses deliberate refusal to decide. The distinction — undecided by neglect vs. undecided by recorded commitment — is real but implicit, and I second-guessed my own deliberately-empty author-secret set for a minute before resolving it. One clause would fix it.

Calibration against examples, not adjectives, is what makes the rubric transferable between stewards.

- **F-15.** The QA scorecard has no n/a convention. - **Q-05.** Tests 29 (instrument wiring) and 30 (vocabulary separation) score *the methodology package itself*, not the world — a world-steward filling the scorecard cannot answer "does every load-bearing protocol name its sweep?" about their fiction. Their presence in the world-QA list is confusing; they belong in a separate package-self-audit subsection (they are good tests — of the docs).

## Verdict

**The system works.** A steward with no authorship stake built a small world from nothing to a QA-passed first stable candidate in one continuous pass, using only the package. More importantly, the instruments *generated* quality rather than merely checking it: the trial log records six design decisions of real value that are directly attributable to a specific instrument prompt (V-03, V-05, V-07, V-08, V-13, V-19) and one genuine self-inflicted contradiction caught by the machinery it was designed for (V-12). The severity mapping works (V-06). The admission/repair jurisdiction boundary works (V-14). The sacred-opacity accountability chain works template-first (V-15). Nothing in the trial suggests an overhaul; everything expensive already earns its keep.

The friction is concentrated in **connective tissue, not substance**: reading-path routing, instrument-to-instrument handoffs, and record lifecycle. Almost every F-item is fixable with sentences, not sections.

## Recommendations for iteration eight, ranked

1. **Close the card-lifecycle gap (F-13 / Q-06).** One doctrine paragraph (likely in `03` or `06`): fact cards are living present-tense records updated in place; superseded wording moves to a history note on the card; reports (propagation, contradiction) are the immutable audit trail. Without this, every long-lived world accumulates stale cards correctable only by replaying reports.
2. **Fix the reading path (F-01, F-16, S-04).** Add `03` to the new-world path (before step 4) and `21` (at step 5, or cite its minimal-use paths from `06`). State explicitly whether `02` is required or enrichment. Resolve the README-vs-`21` change-proposal conflict one way or the other.
3. **Add the batch-admission pattern (F-08) and the card division-of-labor sentence (F-07).** The admission-ledger table I improvised (one row per minor seed: layer, status, tags, op, one consequence) should be blessed as the sanctioned lightweight record — either as a tiny template or as three sentences in `06`. Add one sentence dividing fact-card vs. capability-card duties.
4. **Reconcile the kernel's two definitions (F-04).** Make `05` Phase 1 and `templates/world_kernel.md` agree; decide whether the ordinary-life anchor *person* is real doctrine (if so, some instrument downstream should ask about them again; if not, cut it).
5. **One-page operating card — confirmed by trial.** The outlook's candidate survives contact: a single card with (a) the new-world path with *all* actual dependencies, (b) the label-separation table, (c) the severity map, (d) instrument minimal-use paths, would have prevented most hesitations in this log. `03` + `21` + `06` already contain all its content.
6. **Longitudinal worked example — refined by post-trial check of `19`.** `19`'s Example A already follows one fact through the *causal* lifecycle (propose → decompose → constrain → institutions → contradictions → repair → added temporal/spatial pass), so a new causal walkthrough is not needed. What no example shows is the **records** lifecycle: which instruments get filled at each step, what the fact card looks like before and after a repair supersedes part of it, where the ledger rows and QA profile live. That records-side example is the demonstrative fix for F-07/F-08/F-13 — and this trial's Saltmarrow artifacts (`reports/field-trial-world/`) are a ready-made draft of it. Separately: route the paths to `19` (S-05); it is currently reachable only by browsing.
7. **Small template fixes (one line each):** plural operation fields "operation(s), primary first" (F-09, F-12); rename propagation report's "Severity level" field (F-11); n/a convention in the QA scorecard (F-15); move tests 29–30 to a package-self-audit subsection (Q-05); seed-audit timing note + "initial status = proposed" clarification (F-05, F-06); genre-mode vs. consequence-mode gloss on the kernel template (F-02); decomposition granularity rule (Q-02); an admission station or explicit disposal for aesthetic promises (Q-03); the neglect-vs-commitment clause in `13`'s weak-mystery list (Q-04).
8. **Severity-scale unification: drop it (V-06, F-11).** The seventh iteration's mapping paragraph already solved the substance; only the one cosmetic field label remains. Deeper reinforcement-tail weaving: no trial evidence of need; deprioritize.
