# Admission Ledger

Use when decomposition or a burst of small ideas produces many minor-work facts at once. One row per fact holds everything minor work owes under `06`'s severity-scaled evidence — precise statement, scope, truth layer, status and tags, admission operation(s), and one consequence check — so a fifteen-seed decomposition does not force fifteen cards.

The ledger is a thinking instrument, not a software record. Rows may live in a table, a list, index cards, or a notebook margin; what matters is that every admitted fact has a visible governance record.

## Batch context

- Source of the batch: kernel decomposition, expansion pass, session notes, or other.
- Severity assignment: which clusters are minor or moderate work, and which facts were pulled out for full instruments.
- Gate discipline: `checklists/canon_fact_gate.md` run per fact or per coherent cluster; record which.

## Ledger

|---|---|---|---|---|---|
|  |  |  |  |  |  |

Keep the fact column to the smallest precise version. Keep tags separate from status. When an admission composes operations, record the primary first and the others as component moves — all from `03`'s admission list.

## Promotions

A ledger row is the record for a minor fact only while the fact stays minor. Promote a row to a full card (`templates/canon_fact_card.md`, plus `templates/capability_card.md` or `templates/constraint_card.md` where the type demands) the moment the fact turns out to be load-bearing: when other facts start depending on it, when it acquires canon debt, or when a repair touches it. Note the promotion on the row.

## Debt and follow-ups

Name any canon debt the batch surfaced (missing constraints, deferred passes) and where it was assigned.

## Traceability note

Parent protocol: `06_canon_fact_admission_protocol.md` and `03_truth_layers_and_canon_governance.md`.

This card is a human reasoning instrument, not a database schema or implementation contract. Rows follow `03`'s record lifecycle like any card: update a row in place when canon changes, and note the report that changed it.

## Minimum completion

- Every admitted fact in the batch has a row or a pointer to its full card.
- Every row carries a truth layer, a status separated from its tags, and an admission operation.
- Every row's consequence check says something the world must now answer, not a restatement of the fact.
- Canon debt surfaced by the batch is named and assigned.
