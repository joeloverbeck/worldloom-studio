# Temporal / Timeline Sweep

Serves `09_temporal_and_timeline_protocol.md`. Use when a fact has causal order, latency, age, memory, evidence, branch, or retcon implications.

## Sequence

- [ ] First appearance identified.
- [ ] First discovery identified if knowledge matters.
- [ ] First public awareness identified if public belief matters.
- [ ] First institutional response identified if institutions matter.
- [ ] Prerequisite events precede the fact.
- [ ] Effects follow causes.
- [ ] Existing canon does not ignore the fact retroactively.

## Delay and residue

- [ ] Latency explained.
- [ ] Adoption periods considered.
- [ ] Generational effects considered.
- [ ] Historical fossils listed.
- [ ] Old institutions that should persist are listed.
- [ ] Extinct or obsolete practices are listed.
- [ ] Path dependence recorded.

## Chronology pressure

- [ ] Conflicting chronologies identified.
- [ ] Branch chronology checked.
- [ ] Temporal mysteries have boundaries.
- [ ] Retcon cost assessed.
- [ ] Timeline card needed or not needed decision recorded.

## Version 0.5 completion rule

Parent protocol: `09_temporal_and_timeline_protocol.md`.

A completed sweep must leave `sequence, latency band, competing chronology, and residue`. If the sweep is skipped for a major fact, record why the parent protocol did not apply.

## Red flags

- The checklist is used as a yes/no ritual without recording consequences.
- A box is checked because the author likes the idea rather than because the world now answers it.
- The sweep discovers canon debt but no follow-up is named.
- The sweep changes canon status, constraint tags, or repair operations without using the proper parent protocol.
