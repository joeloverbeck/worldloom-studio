# Spatial / Geographic Sweep

Serves `10_spatial_and_geographic_propagation.md`. Use when a fact changes where things can happen, move, be known, be legal, or leave residue.

## Scope and movement

- [ ] Origin point or region identified.
- [ ] Spatial scope stated.
- [ ] Terrain and climate considered.
- [ ] Routes and corridors identified.
- [ ] Barriers identified.
- [ ] Centers and peripheries considered.
- [ ] Settlement pattern considered.
- [ ] Infrastructure considered.
- [ ] Borders and jurisdictions considered.

## Diffusion and variation

- [ ] Diffusion path described.
- [ ] Diffusion type identified: contagious, hierarchical, relocation, network, or suppressed.
- [ ] Early adopters and holdouts named.
- [ ] Regional variation listed.
- [ ] Negative-space regions explained.
- [ ] Local beliefs and institutions reflect place.
- [ ] Diffusion speed is plausible.

## Residue

- [ ] Map implications recorded, even if no map exists yet.
- [ ] Aesthetic geography residue noted.
- [ ] Smugglers, tolls, forts, dead roads, quarantine markers, local legends, or other barrier residues considered.

## Version 0.5 completion rule

Parent protocol: `10_spatial_and_geographic_propagation.md`.

A completed sweep must leave `diffusion channels, barriers, regional inequality, and spatial residue`. If the sweep is skipped for a major fact, record why the parent protocol did not apply.

## Red flags

- The checklist is used as a yes/no ritual without recording consequences.
- A box is checked because the author likes the idea rather than because the world now answers it.
- The sweep discovers canon debt but no follow-up is named.
- The sweep changes canon status, constraint tags, or repair operations without using the proper parent protocol.
