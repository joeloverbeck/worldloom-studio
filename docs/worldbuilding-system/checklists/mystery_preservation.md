# Mystery Preservation Checklist

Serves `13_contradiction_retcon_and_mystery.md`. Use whenever a contradiction repair, retcon, reveal, adaptation, branch, or evidence update touches a protected unknown.

## Boundary

- [ ] Mystery question is named.
- [ ] Mystery type identified.
- [ ] What is fixed is separated from what is unknown.
- [ ] Author-secret material marked.
- [ ] Deliberately undecided material marked.
- [ ] Forbidden explanations or uses stated.
- [ ] Truth-layer boundaries recorded.

## Evidence and belief

- [ ] Partial knowers listed.
- [ ] False beliefs listed.
- [ ] Evidence available listed.
- [ ] Evidence missing listed.
- [ ] Public, local, elite, mythic, propaganda, lie, honest error, and disputed layers separated where relevant.

## Consequence and reveal

- [ ] Consequences already visible.
- [ ] Institutions, rituals, fears, prices, taboos, or residues caused by the mystery recorded.
- [ ] Reveal permissions stated.
- [ ] Reveal prohibitions stated.
- [ ] The mystery does not excuse missing propagation.
- [ ] The mystery would still matter if partly revealed.
- [ ] The repair does not accidentally solve what should remain protected.

## Version 0.4 completion rule

Parent protocol: `13_contradiction_retcon_and_mystery.md`.

A completed sweep must leave `fixed/unknown boundary, reveal permissions, reveal prohibitions, and consequence record`. If the sweep is skipped for a major fact, record why the parent protocol did not apply.

## Red flags

- The checklist is used as a yes/no ritual without recording consequences.
- A box is checked because the author likes the idea rather than because the world now answers it.
- The sweep discovers canon debt but no follow-up is named.
- The sweep changes canon status, constraint tags, or repair operations without using the proper parent protocol.
