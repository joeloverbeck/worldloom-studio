# Mystery / Wonder Preservation Checklist

Serves `13_contradiction_retcon_and_mystery.md`. Use whenever a contradiction repair, retcon, reveal, adaptation, branch, evidence update, QA repair, or medium translation touches a protected unknown, wonder, sacred opacity, horror, dread, or symbolic-excess effect.

## Boundary

- [ ] Protected effect type identified: governed mystery, wonder/awe/sublimity, sacred opacity, horror/terror/dread, symbolic excess, or hybrid.
- [ ] Puzzle question named if one exists; marked `none` if no puzzle is intended.
- [ ] Mystery type identified where applicable.
- [ ] What is fixed is separated from what is unknown.
- [ ] Author-secret material marked.
- [ ] Deliberately undecided material marked.
- [ ] Forbidden explanations, uses, measurements, mappings, prices, reveals, or profanations stated.
- [ ] Truth-layer boundaries recorded.
- [ ] Preservation boundary recorded separately from canon status, constraint tag, admission decision operation, and repair operation.

## Evidence, belief, and experience

- [ ] Partial knowers listed.
- [ ] False beliefs listed.
- [ ] Evidence available listed.
- [ ] Evidence missing listed.
- [ ] Public, local, elite, mythic, propaganda, lie, honest error, and disputed layers separated where relevant.
- [ ] Awe, dread, reverence, enchantment, taboo, or symbolic-excess experiences named where relevant.
- [ ] Evidence habitat can exist without forcing final explanation.

## Consequence and preservation

- [ ] Consequences already visible.
- [ ] Institutions, rituals, fears, prices, taboos, recurrences, motifs, dread patterns, songs, silence, or residues caused by the effect recorded.
- [ ] Reveal permissions stated.
- [ ] Reveal prohibitions stated.
- [ ] Explanation-pressure operation chosen: reveal, delay, forbid, consecrate, dread-preserve, excess-preserve, or translate.
- [ ] The protected effect does not excuse missing propagation.
- [ ] The protected effect would still matter if partly revealed.
- [ ] The repair does not accidentally solve, secularize, mechanize, domesticate, or decode what should remain protected.
- [ ] The preservation boundary does not hide an ordinary contradiction that should be repaired.


## Version 0.6 sacred-opacity accountability guard

Use this section whenever the protected effect is sacred opacity or adjacent sacred/horror opacity.

- [ ] **Refusal scope:** The exact explanation being refused is stated.
- [ ] **Consequence acceptance:** Costs, residues, ritual boundaries, witness effects, and profanation consequences remain visible and auditable.
- [ ] **Recurrence / boundary:** The steward can tell a deliberate exception from an arbitrary one-off.
- [ ] **Contradiction handling:** Prior and future scenes that violate the boundary are repaired, branched, or rejected.
- [ ] **Critique permission:** The note states which craft questions remain fair: clarity, recurrence, tone, cost, evidence, and audience trust.
- [ ] **Legitimate-vs-cheat comparison:** The entry can be placed beside a cheat version and explain why it is not using “sacred” to dodge work.
- [ ] **Micro-example:** “The mouth in the chapel is sacred” is insufficient. “Its origin is unexplained, but its noon/sunset speech, shadow cost, candle-voice profanation, and witness records are fixed” earns protection.

## Version 0.6 completion rule

Parent protocol: `13_contradiction_retcon_and_mystery.md`.

A completed sweep must leave `protected effect type, fixed/unknown boundary, preservation boundary, reveal permissions, reveal prohibitions, explanation-pressure operation, and consequence record`. If the sweep is skipped for a major fact, record why the parent protocol did not apply.

## Red flags

- The checklist is used as a yes/no ritual without recording consequences.
- A box is checked because the author likes the idea rather than because the world now answers it.
- A mystery is converted into vague wonder to avoid deciding what is fixed.
- Wonder, sacredness, horror, dream logic, or symbolism is used to dodge all consequence.
- A sacred or horrific effect is explained into machinery without an explicit preservation decision.
- The sweep discovers canon debt but no follow-up is named.
- The sweep changes canon status, constraint tags, decision operations, or repair operations without using the proper parent protocol.
