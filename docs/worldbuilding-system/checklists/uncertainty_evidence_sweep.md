# Uncertainty / Evidence Sweep

Serves `14_uncertainty_belief_and_evidence.md`. Use when a claim, rumor, prophecy, testimony, scholarly dispute, legal fact, propaganda campaign, or protected unknown has social consequences.

## Claim and proof

- [ ] Claim separated from fact.
- [ ] Truth layer assigned.
- [ ] Canon status assigned if the claim is in governance.
- [ ] Confidence band assigned.
- [ ] Sources listed.
- [ ] Source access, motive, distortion, copying, translation, coercion, and expertise considered.
- [ ] Counter-evidence listed.
- [ ] What would change confidence is recorded.

## Belief ecology

- [ ] Belief communities identified.
- [ ] Standards of proof differ by institution.
- [ ] Propaganda, lie, honest error, and disputed claim separated.
- [ ] Evidence habitat described.
- [ ] Strategic non-update or ritualized non-update considered.
- [ ] Suppression or controlled evidence considered.

## Mystery boundary

- [ ] Protected unknowns marked.
- [ ] Evidence does not accidentally settle what should remain protected.
- [ ] Needed mystery ledger entry identified.

## Version 0.5 completion rule

Parent protocol: `14_uncertainty_belief_and_evidence.md`.

A completed sweep must leave `claim, confidence behavior, evidence habitat, belief communities, and mystery boundary`. If the sweep is skipped for a major fact, record why the parent protocol did not apply.

## Red flags

- The checklist is used as a yes/no ritual without recording consequences.
- A box is checked because the author likes the idea rather than because the world now answers it.
- The sweep discovers canon debt but no follow-up is named.
- The sweep changes canon status, constraint tags, or repair operations without using the proper parent protocol.
