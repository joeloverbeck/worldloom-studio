# Branching and Collaboration Sweep

Serves `15_branching_versioning_and_collaboration.md`. Use when continuity diverges, material is retired, multiple people govern canon, or a dispute risks silent contradiction.

## Branching

- [ ] Branch reason is strong enough.
- [ ] Parent continuity named.
- [ ] First divergence point named.
- [ ] Changed facts listed.
- [ ] Unchanged anchors listed.
- [ ] Truth-layer and status differences recorded.
- [ ] Timeline, spatial, institutional, character, mystery, and aesthetic differences considered.
- [ ] Merge expectation stated: likely, possible, unlikely, or never.
- [ ] Leakage warnings recorded.
- [ ] Audience/player/reader status stated.

## Collaboration

- [ ] Proposal written.
- [ ] Doctrine at stake named.
- [ ] Reviewer roles assigned as needed: steward, domain reviewer, contradiction challenger, mystery guardian, audience advocate, decider, recorder.
- [ ] Options considered, including constraint, localization, branch, quarantine, supersession, deprecation, and rejection.
- [ ] Consequences compared.
- [ ] Valid prior content preserved where possible.
- [ ] Decision recorded in plain language.
- [ ] Dissent or open questions recorded.
- [ ] Follow-up propagation, branch diff, contradiction report, or QA check assigned.

## Version 0.4 completion rule

Parent protocol: `15_branching_versioning_and_collaboration.md`.

A completed sweep must leave `branch disposition, invariant anchors, leakage warnings, and decision owner`. If the sweep is skipped for a major fact, record why the parent protocol did not apply.

## Red flags

- The checklist is used as a yes/no ritual without recording consequences.
- A box is checked because the author likes the idea rather than because the world now answers it.
- The sweep discovers canon debt but no follow-up is named.
- The sweep changes canon status, constraint tags, or repair operations without using the proper parent protocol.
