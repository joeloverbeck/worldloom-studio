# Canon Fact Gate Checklist

Serves `06_canon_fact_admission_protocol.md`. Use before treating any proposed fact as active canon.

## Identification

- [ ] Fact is stated clearly.
- [ ] Fact type identified.
- [ ] Consequence mode identified.
- [ ] Truth layer assigned.
- [ ] Canon status assigned.
- [ ] Scope stated: time, place, population, branch, institution, medium/adaptation if relevant.
- [ ] Constraint tags listed separately from status.

## Consequence gate

- [ ] Prerequisites listed.
- [ ] Costs identified.
- [ ] Access conditions identified.
- [ ] Constraints or countermeasures listed.
- [ ] First-order consequences listed.
- [ ] Institutions, informal norms, or declared mode-equivalent recurrence/ritual/symbolic pressure considered.
- [ ] Economic effects or mode-equivalent value pressure considered.
- [ ] Enemy/countermeasure response considered.
- [ ] Timeline implications considered.
- [ ] Spatial implications considered.
- [ ] Character psychology implications considered.
- [ ] Truth/evidence implications considered.
- [ ] Mystery, wonder, sacred opacity, horror/dread, and symbolic-excess risks checked.
- [ ] Contradiction risks checked.
- [ ] Aesthetic residue considered.

## Decision context

- [ ] Admission decision operation(s) recorded, primary first: accept, constrain, localize, historicize, reinterpret, institutionalize, price, branch, quarantine, supersede, deprecate, or reject. Composed admissions record component operations as context, all from the admission list (`03`).
- [ ] Repair operation recorded only if a contradiction or instability required repair.
- [ ] Follow-up propagation assigned.
- [ ] Quarantined or rejected alternatives preserved where useful as branch, rumor, lie, or deprecated note.


## Point-of-use non-naturalistic gate

- [ ] For non-naturalistic facts, the gate identifies the consequence route: recurrence, motif, taboo, ritual condition, associative dream return, comic reversal, dread, or symbolic residue.
- [ ] The fact is not accepted merely because it is beautiful, strange, sacred, or funny.
- [ ] **Micro-example:** “The third knock opens any tomb” passes only if thirdness, tomb scope, false knocks, profanation cost, and witness consequences are recorded.

## Completion rule

Parent protocol: `06_canon_fact_admission_protocol.md`.

A completed sweep must leave `admit/constrain/localize/branch/quarantine/reject result plus canon debt`. If the sweep is skipped for a major fact, record why the parent protocol did not apply.

## Red flags

- The checklist is used as a yes/no ritual without recording consequences.
- A box is checked because the author likes the idea rather than because the world now answers it.
- The sweep discovers canon debt but no follow-up is named.
- The sweep changes canon status, constraint tags, or repair operations without using the proper parent protocol.
