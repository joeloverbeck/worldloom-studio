# Constraint Composition Sweep

Serves `08_constraint_composition.md`. Use when multiple limits govern the same fact or a single limit is doing heavy work.

## Constraint inventory

- [ ] Main constrained fact named.
- [ ] Each constraint stated in plain prose.
- [ ] Constraint type identified: access, cost, location, time, population, material, legal, ritual, knowledge, social, biological, ecological, psychological, aesthetic, branch.
- [ ] What each constraint prevents is clear.
- [ ] What each constraint still allows is clear.
- [ ] Constraint cause or mystery boundary named.

## Composition

- [ ] Constraints tested for stacking: do they strengthen each other?
- [ ] Constraints tested for gates: does one unlock or block downstream use?
- [ ] Constraints tested for tradeoffs: what is gained by paying what cost?
- [ ] Constraints tested for thresholds: what changes above or below a line?
- [ ] Constraints tested for sequential failure: what happens if a step is missed?
- [ ] Constraints tested for cancellation: does one make another irrelevant?
- [ ] Constraints tested for contradiction.

## Leakage and residue

- [ ] Bottleneck identified.
- [ ] Loopholes and partial workarounds listed.
- [ ] Enforcement named.
- [ ] Bypass actors named.
- [ ] Countermeasures named.
- [ ] Ordinary-life residue of the limits identified.
- [ ] Institutional or economic residue identified.
- [ ] The constrained fact still remains worth keeping.
