# Institutional, Economic, and Suppression Sweep

Serves `12_institutional_economic_and_suppression_protocol.md`. Use when a fact changes access, scarcity, enforcement, legitimacy, transaction costs, secrecy, exclusion, or power.

## Institutions

- [ ] Action arenas named.
- [ ] Participants, positions, and exclusions identified.
- [ ] Formal rules described.
- [ ] Rules-in-use described.
- [ ] Enforcement described.
- [ ] Legitimacy explained.
- [ ] Institutional lag considered.
- [ ] Institutional competition or jurisdiction fight considered.
- [ ] Failure modes considered.

## Economy

- [ ] Scarcity changes named.
- [ ] Price or value shifts named.
- [ ] Labor changes named.
- [ ] Transaction costs named.
- [ ] Substitutes and complements considered.
- [ ] Surplus capture identified.
- [ ] Externalities identified.
- [ ] Market failures or commons problems considered.
- [ ] Household impacts considered.
- [ ] Black markets considered.

## Suppression and counter-institutions

- [ ] Who benefits if the fact is hidden?
- [ ] Who is punished for knowing, proving, teaching, or using it?
- [ ] What evidence is controlled?
- [ ] What propaganda, euphemism, silence, or controlled vocabulary exists?
- [ ] What counter-institution forms?
- [ ] When does the counter-institution become predatory?
- [ ] What negative space reveals suppression?

## Synthesis

- [ ] One institutional-economic synthesis sentence written.
- [ ] At least one daily-life residue recorded.
- [ ] At least one power conflict recorded.

## Version 0.5 completion rule

Parent protocol: `12_institutional_economic_and_suppression_protocol.md`.

A completed sweep must leave `action arena, rules-in-use, transaction cost, surplus capture, suppression residue, counter-institution`. If the sweep is skipped for a major fact, record why the parent protocol did not apply.

## Red flags

- The checklist is used as a yes/no ritual without recording consequences.
- A box is checked because the author likes the idea rather than because the world now answers it.
- The sweep discovers canon debt but no follow-up is named.
- The sweep changes canon status, constraint tags, or repair operations without using the proper parent protocol.
