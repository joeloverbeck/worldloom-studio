# Operating Card

A one-page quick reference for working stewards. Everything here is condensed from the package `README.md`, `03_truth_layers_and_canon_governance.md`, `06_canon_fact_admission_protocol.md`, and `21_templates_index.md`; on any conflict, those files govern.

## The new-world path

1. Read `01` (doctrine, stopping rules). Keep this card at hand.
2. Fill a lean `templates/world_kernel.md`.
3. Decompose seeds with `05` — split until each seed could be independently rejected without destroying its siblings.
4. Read `03` — the audit and every admission instrument use its vocabulary.
5. Run `checklists/frontloaded_seed_audit.md` (after decomposition, before first admission).
6. Admit with `06` + `checklists/canon_fact_gate.md`: cards for major facts, `templates/admission_ledger.md` rows for minor ones. Check `21` for the lightest instrument that catches the danger.
7. Propagate major facts: `07` + `04` + `templates/propagation_report.md` + `checklists/propagation_sweep.md`. Stop when every high-pressure consequence is answered, scoped out, assigned as debt, or protected.
8. Specialized passes (`08`–`17`) only where the fact applies; run each pass's own sweep.
9. QA with `18` before calling the first version stable.

Stuck on how instruments compose? `19`'s records-lifecycle example walks the full chain. `02` is enrichment, not a required stop.

## The label separations (do not blur)

| Label | Question it answers | Examples |
|---|---|---|
| Truth layer | What kind of truth or claim is this? | objective canon, public belief, lie, disputed claim |
| Canon status | Where is this in governance? | proposed, accepted with constraints, superseded |
| Constraint tag | What limits the fact? | rare, costly, geography-bound, ritual-bound |
| Admission decision operation | What does governance do with the proposal? | accept, constrain, price, institutionalize (`03`) |
| Repair operation | How is an incompatibility treated? | clarify scope, add constraint, split, retcon (`13`) |
| Consequence mode | What form of consequence proves the fact matters? | material/institutional, mythic, sacred, comic |
| Preservation boundary | What must a reveal or repair not destroy? | forbidden explanation, protected dread |

Operations compose within one jurisdiction only: admission with admission, repair with repair — primary first, components as context.

## The severity map

| Admission level (what kind of fact) | Work scale (what work it owes) |
|---|---|
| Level 0 cosmetic / Level 1 local detail | Minor — statement, scope, layer, status, one consequence check; a ledger row is the record |
| Level 2 regional custom or institution | Moderate — minor work + cost/access/constraint check + one domain ripple |
| Level 3 capability or strategic resource | Major — full gate, shock cone, institution/economy pass, contradiction check, debt |
| Level 4 world premise or metaphysical rule | Foundational — major work + temporal/spatial passes, branch, mystery/aesthetic checks, QA profile |
| Level 5 retcon, branch, or repair of established canon | Foundational to catastrophic — plus decision record and rollback/branch plan |

Reports (`propagation_report`, `contradiction_report`) ask for the **work scale**; protocols that trigger on "Level 2 or higher" (`07`, `09`) use the **levels**.

## Minimal instrument paths

- **Tiny local detail:** fact card (or paragraph) + fact gate.
- **Batch of minor facts:** admission ledger, gate per fact or cluster; promote rows that turn load-bearing.
- **Major capability:** gate + fact card + capability card + constraint card (+ constraint sweep if limits interact) + propagation report + sweep; change proposal when alternatives, review, or a recorded decision are needed.
- **Contradiction or retcon:** contradiction report + `13` repair operations; new facts created by the repair go back through admission; mystery-preservation checklist if a protected unknown is touched.
- **Mystery / wonder / sacred opacity:** mystery ledger entry (even when the puzzle question is "none") + mystery-preservation checklist on any later touch.
- **Collaboration dispute:** change proposal + collaboration decision record + branching sweep; branch diff if compromise becomes branch.

## Records rule

Cards are living records: they state the present tense and are updated in place, with outgoing wording moved to a history note. Reports are the audit trail: never edited after the fact. "What is canon today?" = the live cards. "How did it get here?" = the reports in order.

Skipping an instrument is allowed. Silent skipping is not.
