# Method Cards

## Purpose

Method cards are the app-owned guidance layer for PRD #172. A card is a versioned derivation of the methodology package at one decision point. It is app content versioned with the app, not steward world data, and introduces no world-file record type.

## Card Grain

One card exists per decision point: one coherent block of steward-authored material, not a single form field and not a whole flow. Each card carries:

- decision in plain language;
- operative rule in app wording;
- why the method asks;
- what good material looks like;
- upstream package sources;
- derivation version marker;
- guidance depth: `lean`, `standard`, or `full`.

## Derivation Discipline

Cards derive from `docs/worldbuilding-system/` under `docs/principles/domain-fidelity.md` P-1. Each card records upstream sources so package changes create a re-check list. `docs/worldbuilding-system/22_glossary.md` is the naming authority; cards use glossary vocabulary verbatim and do not introduce a synonym layer.

Package file paths are provenance. They may appear in source manifests, audit affordances, or detail panels, but never as the operating instruction. The card text is what the steward and prompt packet receive.

Guidance depth follows `docs/principles/workflow-principles.md` W-2: minor work stays lean, and major-or-higher work earns fuller doctrine and evidence.

## Coverage List

Cards are owed for these guided surfaces:

- Creation: world kernel and seed decomposition.
- Admission: queue/severity selection, minor ledger, full gate, and frontloaded seed audit.
- Constraint Composition: source selection, constrained-fact framing, inventory, composition testing, leakage/residue, outcomes, Prompt-out/skips, close preview, and read-side trail.
- Temporal/Timeline: run entry, temporal questions, date type and granularity, latency/residue, sequence integrity and retrospective insertion, mystery boundaries, outcomes, Prompt-out/skips, close preview, and read-side trail.
- Stage 12: run entry, every lens, outcomes, and close readiness.
- Propagation: run entry, shock-cone orders, domain-atlas sweep, consequence disposition, surfaced proposals, and close.
- Contradiction/Retcon/Mystery: triage, work scale, disposition, repair, retcon cost, repair propagation, mystery preservation, and close.
- QA: pass entry, scorecard, regression profile, pass/fail floor, repair routing, and finalize.
- Setup and workflow map orientation copy.
- Operating-card panel copy, as non-editable server-returned method content with source provenance.

Honestly untested surfaces remain excluded from card authoring per charter T-8: spatial/geographic propagation, agent/character psychology, uncertainty/belief/evidence, narrative/game/transmedia extraction, and aesthetic coherence. They may remain schema-level or generic-record surfaces until field evidence licenses dedicated guided flows.

## Conformance

This spec derives from package doctrine and does not amend it. A contradiction with package authority, glossary vocabulary, W-9 provenance demotion, W-2 severity scaling, or charter T-8 exclusion is surfaced to the steward rather than silently resolved.
